# Project Requirements Analysis

## Assignment Title

`My Learning Journey Website`

## Objective

Build a simple, professional, responsive Next.js App Router website that introduces the student, explains their learning journey, and presents blog sections for frontend, backend, and mobile development.

## Learning Outcomes

- Understand the App Router folder structure
- Create reusable layouts and shared UI
- Use TypeScript in a Next.js project
- Configure Tailwind CSS for responsive design
- Add page-level metadata for SEO
- Organize a clean project for deployment

## Required Pages

- `/`
- `/about`
- `/contact`
- `/blog`
- `/blog/frontend`
- `/blog/backend`
- `/blog/mobile`

## Required Root Layout Features

- `html` language attribute
- Imported Google font
- Global metadata defaults
- Shared Navbar
- Shared Footer
- Global styling and colors
- Responsive page wrapper

## Required Nested Blog Layout Features

- Blog header
- Blog sidebar
- Blog navigation
- Shared content area
- Automatic wrapping for all `/blog/*` routes

## Required SEO Metadata

- Home page title and description
- About page title and description
- Contact page title and description
- Blog overview title and description
- Frontend blog title and description
- Backend blog title and description
- Mobile blog title and description

## Required Responsive Design

- Mobile-first layout
- Collapsible or stacked navigation
- Flexible card grids
- Readable text widths
- No horizontal overflow
- Comfortable tap targets

## Required README Sections

- Project title
- Description
- Features
- Technologies used
- Setup instructions
- Folder structure
- Pages and routes
- App Router concepts demonstrated
- Root layout explanation
- Nested blog layout explanation
- SEO metadata explanation
- Responsive design notes
- Bonus features implemented
- Deployment instructions
- GitHub link placeholder
- Vercel link placeholder

## Deliverables

- Working Next.js project
- Documentation files
- Responsive UI
- Build-ready code

- Custom `404` page
- Loading states with `loading.tsx`
- Reusable `Button` component used throughout the application

## Implementation Plan

1. Inspect the existing scaffold and confirm App Router usage.
2. Create shared config, data, and UI components.
3. Replace the starter page with the required routes.
4. Add the nested blog layout and bonus pages.
5. Write project documentation and review notes.
6. Run lint and build checks, then fix any issues.
